# E-Commerce Order Management Database System

## Week 1 – Understanding the Business Problem

## Project Overview

This project focuses on analyzing the business requirements of an E-Commerce Order Management Database System.
The goal is to understand the business operations of an online shopping platform and prepare a Software Requirement Specification (SRS) document that serves as the foundation for future database design and implementation.

## Objective

* Analyze the business requirements of the system.
* Identify the core entities involved.
* Understand the major business processes.
* Define functional and non-functional requirements.
* Prepare a Software Requirement Specification (SRS).

## Core Entities

* Customer
* Category
* Product
* Supplier
* Order
* Order Details
* Payment
* Shipment
* Review

## Business Processes

* Customer Registration
* Product Management
* Inventory Management
* Order Processing
* Payment Processing
* Shipment Tracking
* Customer Review Management
* Report Generation

## Documents Included

* Requirement Analysis Report
* Business Requirement Document
* Software Requirement Specification (SRS)

##  Technologies Used

* Requirement Analysis
* Business Analysis
* Database Design Concepts
* Documentation

## Conclusion

This project provides a strong foundation for designing an E-Commerce Order Management Database System.
The requirement analysis and SRS created in Week 1 will be used in the upcoming weeks for ER modeling, normalization, SQL implementation, and report generation.
